# iMusic

基于 HarmonyOS 的个性化 AI 云音乐播放器。

本项目以华为开发者案例为基础，面向影音娱乐类应用场景，使用 DevEco Studio、ArkTS 与 ArkUI 开发鸿蒙原生音乐播放器。在基础播放、歌单、推荐、发现、动态、Mine 页面等功能之上，项目重点补强了 AI 音乐助手与用户听歌数据可视化能力，使应用从普通播放器升级为具备“理解用户心情、结合本地音乐知识库推荐歌曲、展示个人音乐画像”的智能音乐应用。

## 项目定位

- 课程方向：以华为开发者案例为基础的鸿蒙影音娱乐类实践项目
- 应用类型：HarmonyOS 原生音乐播放器
- 核心目标：音乐播放、云端曲库、个性化推荐、AI 音乐注释、用户画像可视化
- 开发平台：DevEco Studio
- 主要技术：ArkTS、ArkUI、HarmonyOS Ability、HTTP API、AVPlayer、AppStorage、本地数据建模

## 已完成功能

### 基础音乐播放

- 推荐、发现、动态、播放页、Mine 等主要页面框架
- 歌曲播放、暂停、切歌、页面跳转
- 播放器后台运行权限与音频播放能力
- 歌曲卡片点击播放与全屏播放页联动

### AI 音乐助手“小乐”

- 用户可输入当前心情或听歌场景，例如“我很难过”“我要跑步”“想听周杰伦”
- 系统先在本地音乐知识库中完成候选召回与排序
- 每次请求会向大模型传入多类上下文：
  - 小乐长期记忆：角色、回答风格、展示限制
  - 标签体系摘要：情绪、场景、风格、乐器、音乐因子
  - 歌曲知识库 MD：本地候选曲库及标签信息
- 大模型根据本地候选和知识库生成自然语言推荐解释
- API 不可用时，系统会使用本地推荐结果生成兜底回复，保证演示链路稳定

### 本地音乐知识库

- 建立可扩展的本地歌曲候选库
- 每首歌包含情绪标签、场景标签、风格标签、乐器标签和音乐因子
- 标签体系参考 MTG-Jamendo、musicnn、Essentia 等开源音乐理解项目的常见标注思路
- 推荐流程采用“本地召回优先，大模型负责解释”的方式，避免完全依赖大模型凭空生成

### 个性化推荐

- 根据用户输入识别听歌意图，例如夜晚放松、学习专注、运动充能、怀旧回忆
- 结合本地用户记忆，包括常听歌手、近期歌曲、今日听歌时长、偏好标签
- 对歌曲的能量、愉悦度、节奏、专注度、平静度进行匹配排序
- 推荐结果展示歌曲、歌手、匹配度、标签和简短推荐理由

### Mine 数据可视化

- 今日听歌分钟数、播放歌曲数、常听歌手
- 个人偏好关键词云
- 听歌时段热力图
- 情绪分布与场景分布进度图
- 今日音乐小结，形成轻量级个人音乐报告

## AI 推荐流程

```text
用户输入心情/场景
        ↓
本地意图识别
        ↓
本地音乐知识库排序
        ↓
生成优先候选 + 歌曲知识库上下文
        ↓
调用大模型生成自然语言解释
        ↓
前端展示小乐回复与推荐歌单
```

该设计的重点是让 AI 不直接“猜歌”，而是基于本地歌曲知识库和用户偏好进行解释。这样既能保证推荐结果稳定，也能在答辩中说明歌曲标签、用户记忆和 AI 回复之间的关系。

## 运行方式

1. 使用 DevEco Studio 打开项目根目录 `iMusic-main`
2. 等待 ohpm 依赖同步完成
3. 选择 `entry` 模块
4. 连接鸿蒙模拟器或真机
5. 点击 Run 运行项目

## 答辩展示建议

推荐演示顺序：

1. 展示基础音乐播放器首页和播放页，说明项目完成了完整播放链路
2. 打开 AI 页面，分别输入“我很难过”“我要跑步”“想听周杰伦”，展示小乐会给出不同推荐解释
3. 点击推荐歌曲进入播放页，展示 AI 推荐和播放功能已经联动
4. 打开 Mine 页面，展示今日听歌报告、关键词云、热力图和情绪分布
5. 总结 AI 方案：本地知识库负责召回，大模型负责自然语言解释，用户数据和推荐算法可以复用
